Otters Civ. Revived is a server-oriented Fabric mod that introduces a persistent wallet, chat commands for balances, configurable payouts for breaking certain blocks and killing tagged mobs, and a bundled legacy FPS overlay from an earlier template. It is positioned as early-stage infrastructure toward a broader civ-suite roadmap (jobs, factions, shops).

Commands

Commands register on the logical server (including single-player integrated servers). The in-game analogue of this article is /otter, which prints a short synopsis.

/otter
Lists the mod’s slash commands and points operators to the rewards configuration path. No arguments.
/money
Requires an executing player context. Sends the requesting player’s wallet balance in chat.
/money set <player> <amount>
Sets the target player’s balance to amount (≥ 0). Intended for admin bootstrap and balancing; broader permission gates are planned.

Economy & persistence

Balances are keyed by player UUID and held in memory at runtime with writes to config/fpsmod/wallet.properties. Administrative /money set performs a full replace of the stored balance for that UUID; gameplay rewards call additive credit paths.

The rewards file (rewards.json) configures tags and payout sizes; wallet storage remains under config/fpsmod/.

Passive rewards

Mining and combat payouts are controlled by JSON at config/otters_civ_revived/rewards.json (created with defaults when missing). Defaults include:

  • Block payouts for states in tag otters_civ_revived:currency_blocks (shipped datapack merges stone, basic ores, etc.).
  • Mob payouts for entities in #minecraft:hostile unless changed via entityTag.

Options include rewards per action, cooldowns per player, skipping creative/spectator, dimension blacklist strings, and whether to announce each grant in chat. As of current implementation, ranged or indirect kills (e.g., projectile without direct attribution in the examined damage hook) may not qualify; melee-style direct hits do.

Client features

A legacy FPS counter and pause/inventory overlay toggle persists from the repository’s older template (FpsHudOverlay / HUD config under config/fpsmod/). It is optional ornamentation for the owning client install and unrelated to civ economy semantics.

Operational limits

  • One rewards config directory per launcher instance applies to integrated server usage; wallet file scope follows the same Fabric config root.
  • Tag-based payouts do not imply “worldgen-only” blocks; duplication farming is deferred to roadmap hardening.
  • Future professions may subscribe via the JobsHooks extension seam (currently no-op).
